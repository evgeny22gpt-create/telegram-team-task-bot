# Проверка Apps Script и синхронизации Google → SQLite

Дата проверки: 2026-07-12 (МСК).

## Настройка

- К таблице привязан Apps Script `Telegram Tasks — автообновление UTC`.
- В проекте сохранена копия исходника: `google_apps_script/Code.gs`.
- Используется простой триггер `onEdit(e)` без Deploy.
- Сценарий следит за колонками B, E и F листа `Задачи`.
- При ручном редактировании он записывает ISO 8601 UTC-время в колонку G.

## Контроль триггера

Временная смена категории задачи №1 обновила G2 до нового UTC-времени. После проверки исходная категория `Без категории` была возвращена, и G2 обновилась повторно.

## Сквозная проверка

Статус задачи №1 вручную изменён в Google Таблице:

```text
Новая → В работе
```

Apps Script автоматически установил:

```text
updated_at = 2026-07-12T11:00:06.962Z
```

Первый dry-run:

```text
PULL_TO_SQLITE=1
NO_CHANGE=2
CONFLICT=0
CHANGES_APPLIED=no
```

Применение:

```text
CHANGES_APPLIED=yes
SQLITE_BACKUP=tasks_before_sync_20260712_110043_723546.db
```

После применения SQLite содержит для задачи №1 статус `В работе` и то же `updated_at`. `PRAGMA integrity_check` вернул `ok`.

Повторный dry-run:

```text
PUSH_TO_GOOGLE=0
PULL_TO_SQLITE=0
NO_CHANGE=3
CONFLICT=0
```

Это подтверждает полную цепочку: ручное изменение Google → автоматическое время → dry-run → резервная копия → обновление SQLite → совпадение источников.
