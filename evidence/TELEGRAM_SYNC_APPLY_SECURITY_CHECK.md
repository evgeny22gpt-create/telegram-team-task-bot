# Проверка защищённой команды /sync_apply

Дата проверки: 2026-07-12 (МСК).

## Защита доступа

- ID владельца хранится только в локальном `.env` как `TELEGRAM_OWNER_IDS`.
- В `.env.example` используется вымышленный ID.
- Команда сначала сравнивает `message.from_user.id` со списком владельцев.
- Постороннему пользователю возвращается отказ без раскрытия разрешённых ID.
- Владелец должен отправить точную команду `/sync_apply confirm`.
- Перед применением рекомендуется выполнить безопасный `/sync`.

## Защита данных

- После подтверждения бот повторно получает актуальные данные, а не применяет старый план.
- Конфликт блокирует применение.
- Перед изменениями создаётся проверенная резервная копия SQLite.
- Запрос Google выполняется через `asyncio.to_thread`, не блокируя aiogram.
- Ошибка записывается в локальный журнал без вывода ключа пользователю.

## Проверки

- Разрешённый ID допускается.
- Посторонний и отсутствующий ID отклоняются.
- Ответ при отсутствии изменений не заявляет о создании копии.
- Ответ после применения показывает направления и имя резервной копии.
- Локально настроен ровно один владелец без вывода его ID.
- Прямой запуск применения при трёх совпадающих задачах завершился без записи и без новой копии.

## Ручная проверка защитного барьера

В 14:34 МСК владелец отправил `/sync_apply` без аргумента. Бот не запустил применение и ответил:

```text
Команда может изменить SQLite и Google Таблицу.

Сначала проверьте план: /sync
Для применения отправьте: /sync_apply confirm
```

Скриншот сохранён как `evidence/screenshots/11_sync_apply_confirmation_gate.png`.

## Ручная проверка подтверждённой команды

В 14:36 МСК владелец отправил `/sync_apply confirm`. Бот повторно построил актуальный план и ответил:

```text
Применяю проверенный план синхронизации…

SQLite и Google Таблица уже совпадают. Изменения не требуются ✅
```

Новая резервная копия не создавалась, потому что записей не требовалось. Скриншот сохранён как `evidence/screenshots/12_sync_apply_confirmed_noop.png`.

Ручная проверка защитного барьера и подтверждённого безопасного no-op завершена.
