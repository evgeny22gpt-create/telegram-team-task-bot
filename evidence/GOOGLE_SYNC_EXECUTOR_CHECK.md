# Проверка исполнителя двусторонней синхронизации

Дата проверки: 2026-07-12 (МСК).

## Защитные механизмы

- Без флага `--apply` команда работает только как dry-run.
- Любой конфликт блокирует весь запуск до записи данных.
- Перед реальными изменениями SQLite создаётся резервная копия через `sqlite3.backup`.
- Резервная копия проходит `PRAGMA integrity_check`.
- Входящие задачи применяются одной транзакцией SQLite.
- В Google отправляются только новые или более свежие SQLite-строки.
- Запись использует режим `RAW`, чтобы значения не интерпретировались как формулы.

## Проверка на временных данных

Автоматически подтверждены сценарии:

- dry-run не выполняет записей и не создаёт копию;
- новая Google-версия обновляет SQLite;
- отсутствующая Google-строка добавляется из SQLite;
- перед применением создаётся читаемая резервная копия;
- конфликт блокирует SQLite и Google без частичных изменений.

## Проверка на рабочей таблице

Dry-run:

```text
SYNC_MODE=dry-run
PUSH_TO_GOOGLE=0
PULL_TO_SQLITE=0
NO_CHANGE=3
CONFLICT=0
CHANGES_APPLIED=no
```

Режим `--apply` дал тот же план и не выполнил записей, поскольку все три задачи уже совпадали. Рабочие данные не изменились.
